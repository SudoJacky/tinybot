技能系统

Skills 技能系统

简介

Skills 系统允许你通过 Markdown 文件定义自定义技能，无需编写代码。Agent 在条件匹配时自动遵循定义的工作流程。

技能文件结构

每个技能是一个 SKILL.md 文件，存放在 tinybot/skills/ 目录下。

文件格式

---
name: skill-name
description: 技能描述
always: false
---

# 技能标题

## 触发条件
描述何时触发此技能

## 工作流程
1. 第一步操作
2. 第二步操作
3. ...

## 输出格式
期望的输出格式说明

元数据字段

字段 类型 描述
name string 技能名称（唯一标识）
description string 技能描述
always bool 是否始终加载

技能示例

简单技能

---
name: summarize
description: 总结长文本内容
always: false
---

# 文本总结

## 触发条件
当用户要求总结、概括、提炼文本内容时触发。

## 工作流程
1. 识别需要总结的文本
2. 提取关键信息点
3. 按重要程度排序
4. 生成简洁的总结

## 输出格式
- 使用要点列表
- 每点不超过一句话
- 保持原文的核心信息

复杂技能

---
name: code-review
description: 代码审查技能
always: false
---

# 代码审查

## 触发条件
用户请求代码审查、检查代码质量时触发。

## 检查项
1. **代码风格**
   - 命名规范
   - 格式一致性

2. **潜在问题**
   - 安全漏洞
   - 性能问题
   - 逻辑错误

3. **最佳实践**
   - 是否遵循项目规范
   - 是否有更好的实现方式

## 输出格式
### 问题列表
| 类型 | 文件 | 行号 | 描述 | 建议 |

### 总体评价
- 代码质量评分
- 主要改进建议

技能管理

通过 WebUI

  1. 点击右侧的"Skills"面板
  2. 查看所有技能列表
  3. 点击技能名称查看详情
  4. 使用开关启用/禁用技能

创建新技能

  1. 在技能面板点击 "+" 按钮
  2. 填写技能名称和描述
  3. 编写 SKILL.md 内容
  4. 点击"验证"检查格式
  5. 点击"保存"创建技能

验证技能

点击"验证"按钮检查： - 元数据是否完整 - Markdown 格式是否正确 - 必要字段是否存在

技能状态

状态 描述
启用 技能可用，条件匹配时触发
禁用 技能不可用
始终加载 每次对话都加载此技能

技能目录

默认技能目录：tinybot/skills/

tinybot/skills/

│   └── SKILL.md      # 定时任务
└── ...

技能引用

技能可以引用其他文件作为参考资料：

---
name: opencli-usage
description: OpenCLI 使用指南
---

# OpenCLI 使用

## 参考资料
参见 [commands.md](commands.md) 获取完整命令列表。

## 常用命令
...

引用文件会自动包含在技能上下文中。

最佳实践

  1. 命名规范 - 使用小写字母和连线，如 code-review
  2. 描述清晰 - 让 Agent 能正确判断何时使用
  3. 流程明确 - 分步骤描述工作流程
  4. 格式统一 - 保持一致的输出格式
  5. 引用分离 - 复杂内容放在引用文件中
