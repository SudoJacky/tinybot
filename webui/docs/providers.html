自动检测

设置 provider: auto 时自动检测：

{
  "agent": {
    "model": "gpt-4o",
    "provider": "auto"
  }
}

系统根据模型名称自动选择 Provider。

Agent 配置

{
  "agent": {
    "model": "gpt-4o",
    "temperature": 0.7,
    "max_tokens": 4096,
    "context_window": 128000,
    "max_tool_iterations": 50,
    "reasoning_effort": "medium",
    "timezone": "Asia/Shanghai"
  }
}
参数 描述
model 模型名称
temperature 温度参数 (0-2)
max_tokens 最大输出长度
context_window 上下文窗口大小
max_tool_iterations 最大工具调用次数
reasoning_effort 推理强度：low/medium/high
timezone 时区设置

API Key 管理

直接配置

{
  "providers": {
    "openai": {
      "api_key": "sk-..."
    }
  }
}

环境变量

推荐使用环境变量：

export OPENAI_API_KEY="sk-..."
export DEEPSEEK_API_KEY="sk-..."

系统会自动读取环境变量。

多 Provider 支持

可以配置多个 Provider，Agent 根据模型选择：

{
  "providers": {
    "openai": {
      "api_key": "sk-..."
    },
    "deepseek": {
      "api_key": "sk-..."
    },
    "ollama": {
      "api_base": "http://localhost:11434/v1"
    }
  }
}

使用时指定：

{
  "agent": {
    "model": "deepseek-chat"
  }
}

最佳实践

  1. 环境变量 - API Key 使用环境变量，不要硬编码
  2. 合理温度 - 创意任务用高温度，精确任务用低温度
  3. 工具限制 - 设置合理的 max_tool_iterations
  4. 备用 Provider - 配置多个 Provider 以备切换
