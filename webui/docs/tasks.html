任务调度

任务调度系统 (Agentic DAG)

简介

Tinybot 的 Agentic DAG 任务调度系统可以将复杂任务自动分解为可执行的子任务 DAG（有向无环图），支持智能分解、自动链式执行、并行执行和动态调整。

工作原理

1. 智能分解

LLM 分析用户任务，自动生成带依赖关系的子任务图：

用户任务: "帮我分析这个项目并写一个总结报告"
        ↓
    [分析代码结构]
        ↓
    [提取关键功能] ← [检查依赖关系]
        ↓               ↓
    [生成报告大纲] ← [收集测试信息]
        ↓
    [编写报告内容]
        ↓
    [完成]

2. 自动链式执行

SubAgent 完成子任务后，自动触发依赖它的下一个任务。

3. 并行执行

没有依赖关系的任务可以并行执行，最大化效率。

4. 动态调整

运行过程中可以： - 添加新的子任务 - 移除不再需要的任务 - 修改任务依赖关系

使用示例

基本用法

直接在对话中提出复杂任务，Agent 会自动分解：

用户: 帮我分析 webui 目录的代码结构，生成一份技术文档

Agent 会自动分解为：
1. 扫描 webui 目录结构
2. 分析 HTML/CSS/JS 文件
3. 提取功能模块
4. 生成文档框架
5. 编写各章节内容

监控进度

任务执行时会在 CLI 显示实时进度：

  • 当前执行的任务
  • 已完成的任务
  • 等待中的任务
  • 整体进度百分比

配置选项

在 config.json 中配置任务调度：

{
  "agent": {
    "max_tool_iterations": 50,
    "max_subagent_concurrency": 3
  }
}
参数 描述
max_tool_iterations 单个任务最大工具调用次数
max_subagent_concurrency 最大并行 SubAgent 数量

SubAgent 异步执行

特性

  • 非阻塞执行 - 后台任务不阻塞主对话
  • 并发控制 - 可配置最大并发数防止过载
  • 心跳监控 - 自动检测超时任务，防止僵尸进程
  • 自动通知 - 完成后自动触发主 Agent 总结结果

使用场景

# 主 Agent 继续对话，SubAgent 在后台执行耗时任务
用户: 分析日志文件并找出错误模式

Agent: 我将启动一个后台任务来分析日志...
       [SubAgent 开始执行]
       你可以继续和我聊天，任务完成后我会通知你。

用户: 好的，那帮我查一下最近的提交记录

Agent: [继续处理用户请求...]

[SubAgent 完成] → Agent 收到通知 → 总结分析结果

最佳实践

  1. 任务描述清晰 - 明确说明目标，有助于 Agent 正确分解
  2. 合理并发数 - 根据系统资源设置 max_subagent_concurrency
  3. 监控进度 - 使用 CLI 或 WebUI 查看任务进度
  4. 及时调整 - 发现分解不合理时可手动干预
